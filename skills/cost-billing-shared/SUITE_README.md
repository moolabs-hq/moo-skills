# Cost+Billing Discovery & Instrumentation Suite

A six-skill suite that helps Moolabs customers integrate the generated Moolabs SDKs (`moolabs-hq/moolabs-{py,go,ts}`) into their own repos such that **cost ingest events** and **usage ingest events** flow correctly and optimally. The suite is a framework that **cuts dev time** for the customer's engineering team — without it, a senior engineer spends days-to-weeks grep-ing for handlers, conferring with PM on what to bill, hand-wiring trace context, and getting it wrong; with it, hours-to-days of high-confidence guided work.

Generated by `/agent-skill-creator` on 2026-05-19 from `docs/grooming/2026-05-19-cost-billing-discovery-requirements.md`. Implements the requirements doc's §4.1–§4.6 capabilities with the §10 v1 defaults locked in `cost-billing-shared/v1-decisions-log.md`.

## The seven skills

| Order in pipeline | Slash command | Purpose | Key artifact |
|---|---|---|---|
| 0 | `/cost-billing-bootstrap` | **Run first.** LLM-driven setup: asks for product doc + pricing + repo + telemetry stack + terminology overrides; generates `customer-context/` references every other skill reads. | `.moolabs/customer-context/*` |
| 1 | `/cost-billing-discovery` | Scan customer repo; produce three inventories (cost + usage + output↔input) | `.moolabs/inventory/*.yaml` |
| 2 | `/cost-billing-cloud-bill` | Wire AWS CUR / GCP BigQuery / Azure Cost Management exports; cell ③ findings | `.moolabs/cloud-bill/*.yaml` |
| 3 | `/cost-billing-instrument` | **Codemod that wires SDK calls — the framework's core deliverable** | PR(s) per service |
| 4 | `/cost-billing-drift-lint` | CI step: per-PR drift between code and inventories | `.moolabs/drift/delta-report.yaml` |
| 5 | `/cost-billing-adversarial-review` | 5-phase quality gate; runs 6 times per pipeline | `docs/superpowers/reviews/YYYY-MM-DD-*.md` |
| 6 | `/cost-billing-reconcile` | WAPE/Coverage validation (Moolabs-internal); 3-phase dogfood | `.moolabs/reconcile/*.yaml` |

Plus `cost-billing-shared/` — not a slash-invocable skill; holds the documentation every skill depends on.

## Install — persona-aware

```bash
./cost-billing-shared/install.sh
```

Auto-detects your agent platform (Claude Code, Cursor, Copilot, Windsurf, etc.) and **interactively asks which persona you are**:

1. **finance** — CFO / finance reviewer. Reviews pricing + projected revenue.
2. **product** — Product Manager. Builds output↔input bill of materials.
3. **engineering** — Engineer. Wires SDK calls, runs codemod, reviews drift. (Also **installs CodeGraph** and runs `codegraph init -i` on the customer repo for higher-fidelity code-graph queries.)
4. **all** — Integrator machine; runs the full pipeline end-to-end.

The skill subset installed depends on persona:

| Persona | Skills installed |
|---|---|
| finance | bootstrap, discovery, adversarial-review, shared |
| product | bootstrap, discovery, cloud-bill, adversarial-review, shared |
| engineering | bootstrap, discovery, cloud-bill, instrument, drift-lint, adversarial-review, shared **+ CodeGraph CLI** |
| all | everything (incl. reconcile) **+ CodeGraph CLI** |

Non-interactive:

```bash
./cost-billing-shared/install.sh --persona engineering --repo /path/to/customer/repo
./cost-billing-shared/install.sh --persona finance --no-bootstrap-cta
./cost-billing-shared/install.sh --persona all --project
./cost-billing-shared/install.sh --uninstall          # removes all skills (any persona)
```

Honors `CLAUDE_CONFIG_DIR` for Claude Code installs.

## Quick start (a real integration)

After installing the suite, in your agent session:

```
# 0. FIRST RUN ONLY — bootstrap customer-context (asks 5 questions)
/cost-billing-bootstrap

# 1. Discover what to instrument in the customer's repo
/cost-billing-discovery /path/to/customer/repo

# 2. (Parallel) Wire up cloud-bill exports
/cost-billing-cloud-bill --cloud aws

# 3. (Sequentially, per skill-R/three-role-review.md)
#    The three role-specific docs get generated:
#      - CFO fills usage-events-inventory.yaml + reviews/cfo-spec.md
#      - PM reviews CFO's doc, builds output-input-map.yaml, fills reviews/pm-spec.md
#      - CFO reviews PM's doc (CFO ⇄ PM loop, hard cap 3)
#      - Engineer reviews PM's doc, verifies cost-events-inventory.yaml, fills reviews/engineer-spec.md
#      - PM reviews engineer's doc (Engineer ⇄ PM loop, no cap)

# 4. Holistic adversarial gate
/cost-billing-adversarial-review --phase holistic-pre-codemod

# 5. Run the codemod (the framework's core deliverable)
/cost-billing-instrument /path/to/customer/repo

# 6. Skill R post-codemod review
/cost-billing-adversarial-review --phase post-codemod --pr <pr-url>

# 7. Add CI drift-lint (one-time)
#    Copy cost-billing-drift-lint/assets/github-action.yml to .github/workflows/
```

`/cost-billing-reconcile` runs in Moolabs's CI, not in the customer's pipeline.

## What the suite is and isn't

**It is** a framework that cuts customer-integration time by orchestrating discovery → review → codemod → drift-watch → reconcile across the Moolabs SDK family.

**It isn't:**
- A replacement for the Moolabs SDKs — it WIRES them into customer code.
- A margin-computation system — that's downstream (`moo-acute` + `moo-meter`).
- An admin / config UI — v1 is markdown PR + YAML inventories.

See `cost-billing-shared/anchor-taxonomy.md` for the full out-of-scope list.

## §10 v1 decisions (locked, but revisited at HLD)

The requirements doc has 11 decisions §10 lists before HLD can proceed. The suite locks defensible v1 defaults for all 11 — see `cost-billing-shared/v1-decisions-log.md`. Headline calls:

- **#1 Optimization target:** coverage-first (instrument every emission site with confidence ≥ MEDIUM).
- **#2 Language scope:** Python + TypeScript v1; Go v1.5; Java v2.
- **#3 Acute SDK:** blocking-dependency for cost-only emission; v1 emits cost via OTel.
- **#4 Codemod insert default:** Option B (blocking + documented).
- **#5 Invocation surface:** CLI v1, local-only.
- **#6 Catalog miss:** always surface as cell ④, never silent-skip.
- **#7 Three-role review UX:** markdown PR + YAML inventories + static HTML preview.
- **#8 Skill R:** cross-model reviewer, 5-round cap, no-CRITICAL-or-HIGH stop criterion.
- **#9 Skill C exposure:** engineering-internal only v1.
- **#10 Cell ③ severity:** always-surface, no threshold.
- **#11 R vs 3:** both ship v1.

Each decision is marked "REVISIT AT HLD" in the log so HLD can override.
