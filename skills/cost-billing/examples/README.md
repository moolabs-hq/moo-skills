# Examples — anonymized sample artifacts

This folder contains realistic-but-anonymized output samples that the
Cost+Billing suite generates as it walks through the 4-stage bootstrap chain,
discovery, and instrumentation. Use these as a reference for what "good
output" looks like before you run the suite on your own repo.

| File | Stage / phase | Generated by |
|---|---|---|
| `01-finance.signed.yaml` | Stage 1 (CFO bootstrap) | `/cost-billing-bootstrap-finance` |
| `02-cpo.signed.yaml` | Stage 2 (CPO bootstrap) | `/cost-billing-bootstrap-cpo` |
| `03-team-product-checkout.signed.yaml` | Stage 3 (per-product PM) | `/cost-billing-bootstrap-team-product --product checkout` |
| `04-final-payments-api.signed.yaml` | Stage 4 (per-service engineer) | `/cost-billing-bootstrap-team-engineer --service payments-api` |
| `attribution-bindings.yaml` | Phase 1.6 (Discovery) | `/cost-billing-discovery` |
| `sdk-surface-snapshot.yaml` | Phase 1.5 (Instrument) | `/cost-billing-instrument` |
| `tasks.yaml` | Phase 2c (Instrument) | `/cost-billing-instrument` |

All identifiers are fictional. Real customers will see their own service
names, vendor labels, pricing rates, and tenant identifiers — not these.
